## Parent Initiative

[KAT-I-0007 dev-genie Architecture Pattern and Integration](../initiative.md)

## Goal

Write a manual, step-by-step smoke-test plan that satisfies the initiative's exit criterion: "`dev-genie init` in a fresh repo offers and installs katana cleanly." Checklist must be runnable by a person against the wiring shipped in KAT-T-0022..0025; it does NOT require katana itself to be implemented (the test stops at the "katana offered" step if `/katana-init` does not yet exist).

## Deliverables

- Create: `/Users/danielcassil/Code/katana/.metis/strategies/NULL/initiatives/devgenie-architecture-and-integration/smoke-test.md`

## Architecture & Contracts

The smoke-test doc must contain a numbered checklist with these stages:

1. **Setup** — `mkdir /tmp/katana-smoke && cd /tmp/katana-smoke && git init`. Ensure `dev-genie`, `guardrails`, `audit` plugins are reachable.
2. **Run dev-genie** — invoke `/dev-genie-init`. Confirm the greenfield path is taken (project-detection reports `project_kind == greenfield`).
3. **Walk registry** — confirm dev-genie walks entries 1, 2, 3 in order. Confirm entry 3 is `katana`.
4. **Katana offer** — confirm dev-genie surfaces a yes/no prompt for katana, defaulting per `suggests_katana` from KAT-T-0024.
5. **If katana plugin available** — accept the prompt; confirm `/katana-init` is invoked; confirm `.katana/config.toml` and `.katana/vision.md` are created; confirm MCP server is registered.
6. **If katana plugin NOT available** — confirm the install-check signal correctly reports "not installed" and dev-genie pauses with the documented user message.
7. **Final-state checklist** — confirm each item from `orchestration/SKILL.md` Final-state checklist (including the two new katana lines) is reported.

Each step must be a checkbox. Each step must reference the file the behavior is specified in (absolute path).

## Do / Don't

- Don't require katana to actually be implemented to pass the test — gate the post-`/katana-init` steps behind "if katana plugin available."
- Don't write automation in this task; this is a manual checklist for a human.
- Don't duplicate registry contents — reference them by file + section.
- Don't propose changes to dev-genie or guardrails in this task. If gaps are found during the test, file new tasks instead.
