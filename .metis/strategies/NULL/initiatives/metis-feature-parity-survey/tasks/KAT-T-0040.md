# Survey metis persistence layer (storage, short codes, sync, archive)

## Parent Initiative

[[KAT-I-0008]]

## Goal

Survey the four persistence-layer capabilities of metis — document storage (sqlite + markdown files), short-code allocation/uniqueness, workspace sync, and archive — and produce one decision row per capability (Inherit / Depend / Defer / Skip) with rationale grounded in the Katana vision. This is decision-only; no code is written.

## Deliverables

- `/Users/danielcassil/Code/katana/.metis/strategies/NULL/initiatives/metis-feature-parity-survey/notes/survey-persistence.md`

## Survey criteria

For **each** capability listed under "Capabilities in scope" below, answer all of these questions in order:

1. **What does metis provide?** One-paragraph description grounded in observed behavior (CLI help, on-disk artifacts, sqlite schema).
2. **How is it implemented?** Identify the surface (CLI subcommand, MCP tool name, file convention, sqlite table/index, frontmatter field). Cite the exact name(s).
3. **Evidence sources.** Which files / commands / sqlite queries did you inspect? List paths and command outputs.
4. **Katana need.** MVP-required, Post-MVP, or Never — referenced against the vision document.
5. **Decision.** Exactly one of: Inherit (fork) | Depend (call out to metis) | Defer | Skip.
6. **Rationale.** 2–4 sentences explaining the decision.
7. **If Inherit:** which metis source files / modules / sqlite migrations to copy. If unknown, say "unknown — to investigate in inheritance initiative." Do not guess.
8. **Risks / open questions.** Bullet list. At least one item, even if it's "none identified."

### Capabilities in scope

- **C1: Document storage layout** — `.metis/` directory structure, markdown-with-frontmatter files, the sqlite mirror (`metis.db`).
- **C2: Short codes** — `<PREFIX>-<TYPE>-<NNNN>` allocation, type-scoped uniqueness, the `short_code` column in the `documents` table.
- **C3: Sync** — `metis sync` CLI command; rebuilding the sqlite mirror from on-disk files.
- **C4: Archive** — `metis archive <SHORT_CODE>` CLI command; the `archived/` mirror tree; the `archived` boolean.
