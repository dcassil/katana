# Roll-up master capability table and decisions

## Parent Initiative

[[KAT-I-0008]]

## Goal

Combine the five metis-capability surveys (KAT-T-0040..0044) and the Katana additions catalogue (KAT-T-0045) into a single master capability table — the initiative's primary exit-criteria artifact. Add cross-references so each downstream initiative can find the rows that govern its scope. Mechanical aggregation only — no new decisions are introduced; if a sibling survey is missing or contradictory, flag it rather than fix it.

## Deliverables

- `/Users/danielcassil/Code/katana/.metis/strategies/NULL/initiatives/metis-feature-parity-survey/notes/capability-table.md` — the master table.
- `/Users/danielcassil/Code/katana/.metis/strategies/NULL/initiatives/metis-feature-parity-survey/notes/decision-summary.md` — narrative summary, one paragraph per layer plus a "Katana additions" paragraph.

## Survey criteria

This is an aggregation task. For **each row** copied into the master table, verify and record:

1. **Row source** — which sibling survey file the row came from. Cite the exact heading.
2. **Capability name** — short identifier (e.g., "Persistence / Short codes").
3. **Layer** — Persistence | Rules | Query | Agent-Integration | MCP | Katana Addition.
4. **Decision** — Inherit | Depend | Defer | Skip | (for additions: MVP / Post-MVP).
5. **Rationale (one-line)** — distilled from the sibling survey. Do not invent rationale; quote or paraphrase faithfully.
6. **If Inherit — files/modules to copy** — copy verbatim from sibling survey, or "unknown — see source."
7. **Cross-reference initiatives** — which other KAT-I-* initiatives (per the initiative file's scope list) should consume this row. Best-effort match.
8. **Source-survey conflicts or gaps** — explicitly flag any row where two sibling surveys disagree, or where a referenced detail is missing. Do not resolve; flag.

## Do / Don't

- **Don't** introduce new decisions. If a sibling survey didn't pick Inherit/Depend/Defer/Skip, write "MISSING — see source" in the cell and add the row to "Open questions and conflicts."
- **Don't** rewrite rationale to be more eloquent. Faithful paraphrase only.
- **Don't** write code.
- **Don't** modify the sibling survey files. If you find a typo or gap, flag it in the conflicts column.
- **Don't** transition phases or modify the initiative file.
- **Do** keep the master table in valid markdown table syntax.
- **Do** include every row from every sibling survey — completeness is the point.
- **Do** make consumer-initiative cross-references best-effort; "unknown" is acceptable when no obvious mapping exists.
