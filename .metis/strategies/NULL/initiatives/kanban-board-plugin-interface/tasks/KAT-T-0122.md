# Internal Board Backend

## Parent Initiative
KAT-I-0005 — Kanban Board Plugin Interface.

## Goal
Implement `InternalBoardPort` — a *projection* over the katana doc workspace. The board has no separate state: columns come from `columns_for_workspace()`, cards come from `list_documents()` summaries, status = doc phase. Mutation goes through MCP `transition_phase` only.

## Architecture & Contracts

Inline-restated minimal storage shapes the backend needs (avoid coupling to concrete sqlite types):

```ts
// Borrowed from src/types/document.ts; restated to keep this module
// dependent only on a narrow read interface, not on storage/sqlite.
export interface DocumentSummary {
  short_code: string;
  level: "product-doc" | "epic" | "user-story"
       | "task-high-pass" | "task-low-pass" | "task-ui";
  title: string;
  phase: string;
  parent?: string;
  archived: boolean;
  updated_at: string;
}

export interface InternalBoardDeps {
  /** Read-only: returns all docs honoring filter. Wraps list_documents tool. */
  list_documents(opts: {
    level?: DocumentSummary["level"];
    parent?: string;
    include_archived?: boolean;
  }): Promise<DocumentSummary[]>;

  /** Mutation path: delegates to MCP transition_phase. */
  transition_phase(short_code: string, phase: string): Promise<DocumentSummary>;
}
```

```ts
// src/board/internal.ts
import type {
  BoardPort, BoardPortOptions, BoardSnapshot, BoardFilter, Card, Column,
} from "./port.js";
import { columns_for_workspace, phase_for_column } from "./columns.js";
import type { ShortCode } from "../types/document.js";
import type { InternalBoardDeps, DocumentSummary } from "./internal-deps.js";

export class InternalBoardPort implements BoardPort {
  readonly backend = "internal";
  constructor(
    private readonly deps: InternalBoardDeps,
    private readonly opts: BoardPortOptions,
  ) {}

  async list_board(filter: BoardFilter = {}): Promise<BoardSnapshot> {
    const docs = await this.deps.list_documents({
      level: filter.level,
      parent: filter.parent,
      include_archived: filter.include_archived ?? false,
    });
    const columns: Column[] = columns_for_workspace();
    const cards: Card[] = docs.map((d) => this.#toCard(d));
    return {
      generated_at: new Date().toISOString(),
      backend: this.backend,
      columns,
      cards,
    };
  }

  async get_card(card_id: string): Promise<Card | null> {
    // card_id == short_code for internal backend
    const docs = await this.deps.list_documents({ include_archived: true });
    const d = docs.find((x) => x.short_code === card_id);
    return d ? this.#toCard(d) : null;
  }

  async move_card(card_id: string, to_column_id: string): Promise<Card> {
    const target_phase = phase_for_column(to_column_id);
    const updated = await this.deps.transition_phase(card_id, target_phase);
    return this.#toCard(updated);
  }

  #toCard(d: DocumentSummary): Card {
    return {
      id: d.short_code,
      column_id: d.phase,
      title: d.title,
      short_code: d.short_code as ShortCode,
      level: d.level,
      phase: d.phase,
      parent: d.parent as ShortCode | undefined,
      url: `file://${this.opts.workspace_root}/${d.short_code}.md`,
      updated_at: d.updated_at,
      archived: d.archived,
    };
  }
}
```

A tiny supporting file `src/board/internal-deps.ts` exports `InternalBoardDeps` + `DocumentSummary` types from above, so the wiring layer (KAT-T-0125) can adapt MCP tools without circular imports.

## Do
- Treat the board as a read projection: never write to storage outside `move_card`.
- Use `columns_for_workspace()` from KAT-T-0121 verbatim.
- Inject deps via constructor (testable without sqlite).

## Don't
- Don't open the sqlite db here.
- Don't cache snapshots — every `list_board()` re-reads.
- Don't introduce a `create_card` path; creation goes through `create_document`.
