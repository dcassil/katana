## Parent Initiative

[KAT-I-0002](../initiative.md) — Validation Gates and Completeness Checks.

## Goal

Implement the `template-completeness` gate. It validates that the document body has every REQUIRED section defined for its level (and any CONDITIONAL sections triggered by its subtype), that no section bodies are empty or contain only template boilerplate, and that no `{{var}}` placeholders or italic-instructional `_..._` template prompts survive in REQUIRED section bodies.

### Placeholder detection patterns

```typescript
/** `{{anything}}` templating tokens. */
const HANDLEBARS_RE = /\{\{[^}]+\}\}/;
/** Italic instructional prose used in templates: `_..._` on its own line. */
const INSTRUCTIONAL_RE = /^_[^_].*_$/m;
/** Bare `TBD`, `REQUIRED`, `CONDITIONAL`, `OPTIONAL` markers on their own line. */
const MARKER_RE = /^(TBD|REQUIRED|CONDITIONAL(?:\s*\([^)]*\))?|OPTIONAL)\s*$/m;
```

### Reason codes

| Code | When |
|------|------|
| `template.section-missing` | A required (or triggered conditional) section heading is absent | error |
| `template.section-empty` | The section is present but its body is empty (after trim) | error |
| `template.placeholder-handlebars` | Body contains `{{...}}` | error |
| `template.placeholder-instructional` | Body contains an instructional italic line | error |
| `template.placeholder-marker` | Body contains a bare `REQUIRED`/`TBD`/`CONDITIONAL`/`OPTIONAL` marker | error |

### Implementation

```typescript
export const templateCompletenessGate: Gate = {
  name: "template-completeness",
  evaluate(doc: KatanaDocument, _ctx: GateContext): GateResult {
    const reasons: GateReason[] = [];
    const level = doc.frontmatter.level;
    if (!level || !SECTIONS_BY_LEVEL[level]) {
      return { gate: "template-completeness", ok: true, reasons }; // upstream gate handles invalid level
    }
    const subtype = (doc.frontmatter.subtype ?? null) as string | null;
    const rules = SECTIONS_BY_LEVEL[level]!;
    const sectionByHeading = new Map<string, DocumentSection>();
    for (const s of doc.sections) {
      if (s.depth === 2) sectionByHeading.set(s.heading, s);
    }

    for (const rule of rules) {
      const triggered = rule.required || rule.whenSubtype.includes(subtype);
      if (!triggered) continue;
      const section = sectionByHeading.get(rule.heading);
      if (!section) {
        reasons.push({
          code: "template.section-missing",
          message: `Section "## ${rule.heading}" is missing.`,
          severity: "error",
          pointer: `section:${rule.heading}`,
        });
        continue;
      }
      const body = section.body.trim();
      if (body === "") {
        reasons.push({
          code: "template.section-empty",
          message: `Section "## ${rule.heading}" is empty.`,
          severity: "error",
          pointer: `section:${rule.heading}`,
        });
        continue;
      }
      if (HANDLEBARS_RE.test(body)) {
        reasons.push({
          code: "template.placeholder-handlebars",
          message: `Section "## ${rule.heading}" still contains {{...}} placeholders.`,
          severity: "error",
          pointer: `section:${rule.heading}`,
        });
      }
      if (INSTRUCTIONAL_RE.test(body)) {
        reasons.push({
          code: "template.placeholder-instructional",
          message: `Section "## ${rule.heading}" still contains template instructional prose (_..._).`,
          severity: "error",
          pointer: `section:${rule.heading}`,
        });
      }
      if (MARKER_RE.test(body)) {
        reasons.push({
          code: "template.placeholder-marker",
          message: `Section "## ${rule.heading}" still contains a bare REQUIRED/CONDITIONAL/OPTIONAL/TBD marker.`,
          severity: "error",
          pointer: `section:${rule.heading}`,
        });
      }
    }

    const ok = !reasons.some((r) => r.severity === "error");
    return { gate: "template-completeness", ok, reasons };
  },
};
```

## Do / Don't

- DO copy `SECTIONS_BY_LEVEL` verbatim. The headings must match the templates byte-for-byte.
- DO check only depth-2 (`##`) sections; subsection content (`###`) is part of its parent body.
- DO trim body before emptiness check.
- DON'T validate frontmatter here (handled by `frontmatter-schema`).
- DON'T validate that `Child Tasks` has both `### High-pass` and `### Low-pass` subheadings — that's `decomposition-readiness`.
- DON'T add new dependencies.
- DON'T treat instructional `_..._` as valid content even if the executor "filled in" surrounding text — if the regex matches anywhere in the body it's a fail.
