## Parent Initiative

[KAT-I-0001 — Document Templates and Schema](../initiative.md)

## Goal

Author the Task[low-pass] template. Low-pass tasks are run by a cheap/fast model whose only job is to fill in placeholder bodies in scaffolded files produced by a high-pass task. The template removes architectural decisions from the executor.

## Deliverables

- `templates/task-low-pass/_base.md` (new)

## Architecture & Contracts

Frontmatter: per KAT-T-0001 task-low-pass field set. REQUIRED: `parent: KAT-US-####`, `pass: low`, `model_tier: cheap`, `scaffold_task: KAT-TH-####`.

Body sections (in order, all REQUIRED unless marked OPTIONAL):

1. `## Parent User Story` — one-line link.
2. `## Scaffold Source` — REQUIRED. One-line link to `scaffold_task` short code; restate that all files, exports, signatures, and types come from that task and MUST NOT be changed.
3. `## Files to Implement` — bulleted list of relative paths copied from the high-pass `## Files to Create or Modify`. Each entry lists which exported symbols still have placeholder bodies.
4. `## Implementation Notes` — short, scoped guidance per file or per symbol. No new architecture.
5. `## Test Hooks` — OPTIONAL. Pointer to test files or fixtures the implementation must satisfy.
6. `## Allowed Edits` — REQUIRED. Explicit allowlist (e.g. function bodies, private helpers within scaffolded files) and explicit denylist (signatures, exports, types, file structure, new files).
7. `## Acceptance` — checklist: every placeholder body replaced; no signature or export changed; no new files created; tests/lint pass.
8. `## Cross-References` — short codes only.

Italic one-line hint under each heading.

## Do / Don't

- DO make the denylist in `## Allowed Edits` exhaustive — this is the template's main job.
- DO insist that signature/export/type changes are out of scope: if needed, the executor must escalate, not edit.
- DON'T allow creation of new files in the template's allowed-edits section.
- DON'T re-specify the scaffold contract; reference the high-pass task.
- DON'T add architecture/design sections — there are none at this pass.

## Acceptance

- [ ] File exists at `templates/task-low-pass/_base.md`.
- [ ] All 8 sections present in order with italic hints.
- [ ] Frontmatter declares `pass: low`, `model_tier: cheap`, and `scaffold_task` as required.
- [ ] `## Allowed Edits` contains both an allowlist and a denylist with at least four denylist entries (signatures, exports, types, new files).
- [ ] `## Scaffold Source` explicitly forbids changing scaffold-task outputs.
