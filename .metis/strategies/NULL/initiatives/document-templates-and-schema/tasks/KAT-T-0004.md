## Parent Initiative

[KAT-I-0001 — Document Templates and Schema](../initiative.md)

## Goal

Author the Epic template family: one base template + three subtype overlays (`architecture`, `major-feature`, `ui`). Same overlay model as KAT-T-0003.

## Deliverables

- `templates/epic/_base.md` (new)
- `templates/epic/architecture.md` (new)
- `templates/epic/major-feature.md` (new)
- `templates/epic/ui.md` (new)

## Architecture & Contracts

### `_base.md`

Frontmatter: per KAT-T-0001 epic field set. `parent` is REQUIRED and must be a `KAT-PD-####`.

Body sections (in order):

1. `## Parent Product Doc` — REQUIRED. One-line link to parent short code; do NOT restate the product doc's content.
2. `## Summary` — REQUIRED. 1–3 sentences.
3. `## Scope` — REQUIRED.
4. `## Out of Scope` — REQUIRED.
5. `## Architecture Decisions` — CONDITIONAL (subtype = architecture).
6. `## Affected Modules / Surfaces` — CONDITIONAL (subtype = architecture | major-feature).
7. `## Feature Behavior` — CONDITIONAL (subtype = major-feature).
8. `## UX Flows & States` — CONDITIONAL (subtype = ui).
9. `## Component Inventory` — CONDITIONAL (subtype = ui).
10. `## Risks & Open Questions` — OPTIONAL.
11. `## Exit Criteria` — REQUIRED. Bulleted, mechanically checkable.
12. `## Child User Stories` — REQUIRED. List of `KAT-US-####`; may be empty pre-decomposition.

Each section heading followed by an italic one-line hint.

### Overlay files

Same structure as KAT-T-0003 overlays (`## Required Sections`, `## Conditional Sections (...)`, `## Forbidden Sections`).

- `architecture.md` — adds `## Architecture Decisions`, `## Affected Modules / Surfaces`. UX sections forbidden.
- `major-feature.md` — adds `## Feature Behavior`, `## Affected Modules / Surfaces`. Architecture-decision section optional, UX sections forbidden.
- `ui.md` — adds `## UX Flows & States`, `## Component Inventory`. Architecture sections forbidden.

## Do / Don't

- DO link parent product doc by short code in `## Parent Product Doc`; one line, no restated content.
- DON'T re-document frontmatter fields; reference KAT-T-0001.
- DON'T introduce new subtype names beyond the three above.
- DON'T duplicate base sections in overlays.

## Acceptance

- [ ] All four files exist at the paths above.
- [ ] `_base.md` contains all 12 sections in order with REQUIRED/CONDITIONAL labels and italic hints.
- [ ] Each overlay has exactly the three required sections and references base headings only.
- [ ] Frontmatter conforms to KAT-T-0001 epic spec; `parent` documented as `KAT-PD-####`.
