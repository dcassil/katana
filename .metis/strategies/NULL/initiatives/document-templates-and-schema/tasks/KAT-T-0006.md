## Parent Initiative

[KAT-I-0001 — Document Templates and Schema](../initiative.md)

## Goal

Author the Task[high-pass] template. High-pass tasks are run by a strong model and produce a scaffold (placeholder files, type defs, contract comments) that a low-pass task will later fill in. The template must structure the task around scaffold output, not implementation.

## Deliverables

- `templates/task-high-pass/_base.md` (new)

## Architecture & Contracts

Frontmatter: per KAT-T-0001 task-high-pass field set. REQUIRED: `parent: KAT-US-####`, `pass: high`, `model_tier: strong`.

Body sections (in order, all REQUIRED unless marked OPTIONAL):

1. `## Parent User Story` — one-line link. Do not restate the story.
2. `## Goal` — one paragraph: what scaffold this task produces.
3. `## Files to Create or Modify` — bulleted list of relative paths with one-line purpose each.
4. `## Scaffold Contract` — for each file:
   - declared exports (names + types/signatures)
   - required JSDoc / contract comments at function level
   - placeholder body convention (e.g. `throw new Error("TODO: implement (KAT-TL-XXXX)")`)
5. `## Types & Interfaces` — typed blocks (TS or pseudo-code) lifted from the parent story's interface contract.
6. `## Cross-References` — short codes only (parent story, sibling high-pass tasks, ADRs).
7. `## Do / Don't for Executor` — OPTIONAL but recommended.
8. `## Acceptance` — checklist: scaffold files compile/parse, exports match contract, no implementation logic in bodies, placeholder convention used uniformly.
9. `## Hand-off to Low-Pass` — REQUIRED. The short code of the low-pass task that will consume this scaffold (`KAT-TL-####`); may be `TBD` until decomposition.

Italic one-line hint under each heading.

## Do / Don't

- DO frame every section around producing a scaffold, NOT working code.
- DO mandate placeholder bodies (`throw new Error("TODO: ...")`) so low-pass executor cannot accidentally rely on partial implementations.
- DON'T include sections about test execution, deployment, or full implementation — those belong to low-pass.
- DON'T redefine frontmatter; reference KAT-T-0001.
- DON'T add JSDoc requirements to internal helpers in this template (only to exported contract surface).

## Acceptance

- [ ] File exists at `templates/task-high-pass/_base.md`.
- [ ] All 9 sections present in order with italic hints.
- [ ] Frontmatter declares `pass: high` and `model_tier: strong` as required.
- [ ] `## Scaffold Contract` enumerates the three sub-bullets above.
- [ ] `## Hand-off to Low-Pass` documents the `KAT-TL-####` reference.
