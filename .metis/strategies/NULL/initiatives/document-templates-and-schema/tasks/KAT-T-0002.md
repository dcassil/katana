## Parent Initiative

[KAT-I-0001 — Document Templates and Schema](../initiative.md)

## Goal

Document the short-code convention for Katana documents: prefix, type code, numeric tail, regex, allocation rules, and uniqueness scope. This spec is the single source of truth referenced by KAT-T-0001 and every template task.

## Deliverables

- `docs/schema/short-codes.md` (new)

## Architecture & Contracts

`docs/schema/short-codes.md` sections, in order:

1. `# Katana Short Codes`
2. `## Format` — `<PROJECT>-<TYPE>-<NNNN>`, all uppercase, zero-padded 4-digit tail.
3. `## Type Codes`

   | Doc type        | Code | Example       |
   | --------------- | ---- | ------------- |
   | product-doc     | PD   | KAT-PD-0001   |
   | epic            | E    | KAT-E-0001    |
   | user-story      | US   | KAT-US-0001   |
   | task-high-pass  | TH   | KAT-TH-0001   |
   | task-low-pass   | TL   | KAT-TL-0001   |
   | task-ui         | TU   | KAT-TU-0001   |
   | adr (optional)  | A    | KAT-A-0001    |

4. `## Regex` — canonical regex: `^[A-Z][A-Z0-9]{1,9}-(PD|E|US|TH|TL|TU|A)-\d{4}$`.
5. `## Allocation Rules` — bulleted: monotonically increasing per type; never reused; archived docs keep their code; project prefix configurable per workspace (default `KAT`).
6. `## Uniqueness` — short codes are unique workspace-wide across all types; numeric tails are unique only within their type.
7. `## Examples` — at least one example per type plus one invalid example with explanation.

## Do / Don't

- DO use the exact type codes above; downstream templates will hardcode these.
- DON'T introduce variant suffixes (e.g. `KAT-E-0001-UI`); variants are encoded in the `subtype` frontmatter field per KAT-T-0001.
- DON'T specify how codes are allocated programmatically (counter storage, race conditions) — that's MCP/storage scope (KAT-I-0004).
- DON'T duplicate any frontmatter-field definitions; reference KAT-T-0001.

## Acceptance

- [ ] File exists at `docs/schema/short-codes.md`.
- [ ] Type-codes table includes all seven entries above.
- [ ] Regex provided and matches every example.
- [ ] At least one invalid example with rejection reason.
- [ ] Project prefix is documented as configurable.
