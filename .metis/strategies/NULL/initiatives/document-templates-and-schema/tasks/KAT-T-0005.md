## Parent Initiative

[KAT-I-0001 — Document Templates and Schema](../initiative.md)

## Goal

Author the User Story template family: one base template + three subtype overlays (`architecture`, `interface-contract`, `ui`). Same overlay model as preceding tasks.

## Deliverables

- `templates/user-story/_base.md` (new)
- `templates/user-story/architecture.md` (new)
- `templates/user-story/interface-contract.md` (new)
- `templates/user-story/ui.md` (new)

## Architecture & Contracts

### `_base.md`

Frontmatter: per KAT-T-0001 user-story field set. `parent` REQUIRED and must be `KAT-E-####`.

Body sections (in order):

1. `## Parent Epic` — REQUIRED. One-line link.
2. `## Story` — REQUIRED. "As a … I want … so that …" form.
3. `## Acceptance Criteria` — REQUIRED. Bulleted, mechanically checkable.
4. `## Architectural Constraints` — CONDITIONAL (subtype = architecture).
5. `## Interface Contract` — CONDITIONAL (subtype = interface-contract). Must include a typed signature block (TS/pseudo-code).
6. `## Data Shapes` — CONDITIONAL (subtype = interface-contract).
7. `## UI States & Interactions` — CONDITIONAL (subtype = ui).
8. `## Edge Cases` — REQUIRED.
9. `## Out of Scope` — REQUIRED.
10. `## Child Tasks` — REQUIRED. Two subsections: `### High-pass` (`KAT-TH-####`) and `### Low-pass` (`KAT-TL-####`); may be empty pre-decomposition.

### Overlay files

Same shape as preceding overlays.

- `architecture.md` — adds `## Architectural Constraints`. Interface-contract and UI sections forbidden.
- `interface-contract.md` — adds `## Interface Contract`, `## Data Shapes`. UI sections forbidden.
- `ui.md` — adds `## UI States & Interactions`. Architectural/contract sections forbidden.

## Do / Don't

- DO require typed signature in `## Interface Contract` for that subtype — this is what the high-pass task scaffolds against.
- DON'T restate epic content in `## Parent Epic`.
- DON'T merge high-pass and low-pass into one subsection; keep them separate per spec.
- DON'T redefine frontmatter fields.

## Acceptance

- [ ] All four files exist.
- [ ] `_base.md` contains the 10 sections in order with REQUIRED/CONDITIONAL labels and italic hints.
- [ ] `## Child Tasks` has the two named subsections.
- [ ] Each overlay file has the three standard overlay sections and references base headings only.
- [ ] `interface-contract.md` notes that `## Interface Contract` requires a typed signature.
