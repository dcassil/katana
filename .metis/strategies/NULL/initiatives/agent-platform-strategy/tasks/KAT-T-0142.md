# Claude Code Adapter: Plugin Scaffold and Slash Commands

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Implement the directory scaffold and slash-command registration pieces of the Claude Code adapter. Specifically: create `.claude/plugins/katana/` under the host repo with `plugin.json`, a `commands/` directory containing one `.md` file per universal command, and a `skills/` directory with the katana skill definition. Wire `registerCommand()` to write a single command file.

## Deliverables
- `src/platform/claude-code/index.ts` — `ClaudeCodeAdapter` class implementing `PlatformAdapter`. This task implements `id`, `install()` (plugin scaffold + commands portion only — MCP and CLAUDE.md handled by KAT-T-0143/0144), and `registerCommand()`.
- `src/platform/claude-code/templates/plugin.json.tmpl`
- `src/platform/claude-code/templates/commands/katana-decompose.md.tmpl`
- `src/platform/claude-code/templates/commands/katana-work.md.tmpl`
- `src/platform/claude-code/templates/commands/katana-board.md.tmpl`
- `src/platform/claude-code/templates/commands/katana-validate.md.tmpl`
- `tests/platform/claude-code/scaffold.test.ts`

## Architecture & Contracts
Filesystem layout produced under `<workspaceRoot>/.claude/plugins/katana/`:
```
plugin.json
commands/
  katana-decompose.md
  katana-work.md
  katana-board.md
  katana-validate.md
```
`plugin.json` shape:
```json
{ "name": "katana", "version": "0.1.0", "description": "Katana workflow plugin", "commands": ["katana-decompose","katana-work","katana-board","katana-validate"] }
```
Each command `.md` file is a slash-command spec: frontmatter (`name`, `description`, `argument-hint`) + body that instructs the agent to invoke the corresponding MCP tool from KAT-A-0001 (`decompose_document`, etc.).

Idempotency: read existing file, hash, only write if different; record `created`/`updated`/`skipped` in `WrittenFile.action`.

## Do / Don't
- DO use templates in `templates/` loaded via `fs.readFileSync` at install time, with simple `{{var}}` substitution.
- DO honor `opts.dryRun` and `opts.force`.
- DON'T register the MCP server here (KAT-T-0143).
- DON'T modify `CLAUDE.md` here (KAT-T-0144).

## Acceptance
- `install()` against a temp dir produces the four command files and `plugin.json` exactly matching golden fixtures.
- Re-running `install()` reports all files as `skipped`.
- `registerCommand({id:"katana-foo", ...})` writes `commands/katana-foo.md`.
