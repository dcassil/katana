# Claude Code Adapter: MCP Server Wiring

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Add MCP-registration behavior to the Claude Code adapter. During `install()` the adapter writes/updates a `.mcp.json` file at the workspace root that declares the katana MCP server (transport: stdio, command from `InstallOptions.mcpCommand`). Per `install-paths.md`, katana registers its own MCP server during `/katana-init`; this task implements that step inside the adapter.

## Deliverables
- Extension to `src/platform/claude-code/index.ts` adding `writeMcpConfig()` private method called from `install()`.
- `src/platform/claude-code/templates/mcp.json.tmpl`
- `tests/platform/claude-code/mcp-wiring.test.ts`

## Architecture & Contracts
Target path: `<workspaceRoot>/.mcp.json`.
Shape (merge if file already exists — preserve foreign servers):
```json
{
  "mcpServers": {
    "katana": {
      "command": "{{mcpCommand}}",
      "args": ["{{mcpArgs joined}}"],
      "transport": "stdio"
    }
  }
}
```
Merge rule: parse existing JSON; set `mcpServers.katana = {...}`; preserve all other keys and other server entries verbatim. If parse fails, fail loudly with a diagnostic (do not overwrite). Set `InstallReport.mcpRegistered = true` on success.

Universal note: per KAT-A-0001 the MCP server is `@modelcontextprotocol/sdk` stdio. Default `mcpCommand` in tests = `"npx"`, `mcpArgs = ["-y","katana-mcp"]`.

## Do / Don't
- DO preserve existing `mcpServers` entries when merging.
- DO support `--force` to overwrite a malformed `katana` entry.
- DON'T shell out to `claude mcp add` — write the file directly so the same logic works in non-interactive installs.
- DON'T touch `.claude/plugins/` here.

## Acceptance
- Fresh repo: `install()` creates `.mcp.json` with only the katana server.
- Existing `.mcp.json` with `mcpServers.foo`: after install, both `foo` and `katana` present.
- Re-install: file unchanged, `WrittenFile.action === "skipped"`.
- `mcpRegistered` true in returned `InstallReport`.
