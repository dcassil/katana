# Shared FS Utilities (Marker IO, JSON Merge, Idempotent Write)

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Extract the three repeated mechanics every adapter needs into a shared module so adapters stay declarative and behavior is uniformly tested. Used by KAT-T-0142..0146 and KAT-T-0150.

## Deliverables
- `src/platform/_shared/fs.ts` with the functions below.
- `src/platform/_shared/markers.ts`
- `src/platform/_shared/json-merge.ts`
- `tests/platform/_shared/*.test.ts` (one per file).

## Architecture & Contracts
```ts
// fs.ts
export interface IdempotentWriteResult { action: "created"|"updated"|"skipped"; bytes: number; }
export function idempotentWrite(path: string, contents: string|Buffer, opts?: { force?: boolean; dryRun?: boolean }): Promise<IdempotentWriteResult>;
export function readUtf8OrNull(path: string): Promise<string|null>;

// markers.ts
export const KATANA_BEGIN = "<!-- katana:begin -->";
export const KATANA_END   = "<!-- katana:end -->";
export function injectMarkerBlock(existing: string|null, block: string): string;   // create/replace
export function stripMarkerBlock(existing: string): string;                         // for uninstall
export function hasDuplicateMarkers(existing: string): boolean;

// json-merge.ts
export function mergeMcpServers(existingJson: string|null, name: string, entry: object): string;
export function removeMcpServer(existingJson: string, name: string): string|null;   // null = delete file
```
Rules:
- `idempotentWrite` hashes existing contents (sha256) and skips when equal.
- `injectMarkerBlock` always emits exactly one trailing newline after the end marker.
- `mergeMcpServers` preserves key order on round-trip; throws `MalformedMcpJsonError` on parse failure unless `force` is later applied at the adapter level.

## Do / Don't
- DO keep these utilities adapter-agnostic.
- DO use Node `fs/promises` and `crypto` only — no third-party deps.
- DON'T leak adapter-specific concepts (CLAUDE.md, .cursor) here.
