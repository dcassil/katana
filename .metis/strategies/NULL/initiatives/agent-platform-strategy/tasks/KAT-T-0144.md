# Claude Code Adapter: CLAUDE.md Block Injection

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Implement `generateAgentDoc()` for the Claude Code adapter and call it from `install()`. The adapter injects (or updates) a katana-owned, marker-bracketed block in `<workspaceRoot>/CLAUDE.md` describing the four universal commands and pointing to the workspace root.

## Deliverables
- Implementation of `generateAgentDoc()` in `src/platform/claude-code/index.ts`.
- `src/platform/claude-code/templates/claude-md-block.md.tmpl`
- `tests/platform/claude-code/agent-doc.test.ts`

## Architecture & Contracts
Markers (constants exported from `src/platform/port.ts` or adapter-local):
```
<!-- katana:begin -->
... content ...
<!-- katana:end -->
```
Behavior:
- File missing -> create `CLAUDE.md` containing only the marker block.
- File exists, no markers -> append marker block at EOF, preceded by one blank line.
- File exists, markers present -> replace content between markers in-place; preserve surrounding text byte-for-byte.
- Multiple marker pairs -> error with rule_id `katana.claude-md.duplicate-markers`.

Block content (rendered from template, variables: `katanaRoot`, command list):
```
## Katana Workflow

This repo uses katana. Workspace: `{{katanaRoot}}`.

Universal commands:
- /katana-decompose <parent>
- /katana-work <task>
- /katana-board
- /katana-validate <short-code>

Always invoke through the katana MCP server (`mcpServers.katana` in `.mcp.json`).
```

## Do / Don't
- DO use exact byte-level markers; do not trim whitespace inside.
- DO preserve text outside markers verbatim.
- DON'T touch `AGENTS.md` here (Codex adapter owns it).
- DON'T duplicate the block.

## Acceptance
- Three fixture cases (missing / no markers / existing markers) all match golden output.
- Idempotent on re-run.
- Returns `WrittenFile` with correct `action`.
