# OpenAI/Codex Adapter (Minimal Stub)

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Implement a minimal third adapter to prove the abstraction is not Claude+Cursor-specific. Writes an `AGENTS.md` katana block at workspace root and a `.katana/agents-manifest.json` describing the universal commands in a platform-neutral, machine-readable form.

## Deliverables
- `src/platform/openai-codex/index.ts`
- `src/platform/openai-codex/templates/agents-block.md.tmpl`
- `src/platform/openai-codex/templates/manifest.json.tmpl`
- `tests/platform/openai-codex/install.test.ts`

## Architecture & Contracts
Files written:
- `<workspaceRoot>/AGENTS.md` — marker-bracketed block (markers: `<!-- katana:begin --> ... <!-- katana:end -->`) listing the four universal commands and the path to `.katana/agents-manifest.json`.
- `<katanaRoot>/agents-manifest.json`:
```json
{
  "schemaVersion": 1,
  "platform": "openai-codex",
  "mcp": { "command": "{{mcpCommand}}", "args": {{mcpArgs}}, "transport": "stdio" },
  "commands": [
    { "id": "katana-decompose", "mcpTool": "decompose_document", "description": "..." },
    { "id": "katana-work",      "mcpTool": "list_documents",     "description": "..." },
    { "id": "katana-board",     "mcpTool": "list_documents",     "description": "..." },
    { "id": "katana-validate",  "mcpTool": "validate_document",  "description": "..." }
  ]
}
```
`registerCommand(spec)` appends to `commands[]` (dedupe by `id`). `registerRule()` is a no-op returning `[]` with a warning recorded in `InstallReport.warnings`. `generateAgentDoc()` writes/updates the marker block in `AGENTS.md`. `uninstall()` removes the marker block and deletes `agents-manifest.json`.

## Do / Don't
- DO keep this stub small — the goal is to prove the abstraction.
- DO follow the same marker idempotency rules as the Claude Code adapter.
- DON'T invent OpenAI-specific config files; `AGENTS.md` + manifest is the entire surface.
- DON'T attempt MCP auto-registration (Codex has no standard mechanism — manifest documents the command for the user).

## Acceptance
- `install()` on a fresh dir produces `AGENTS.md` + `agents-manifest.json` matching fixtures.
- `mcpRegistered: false` in report; warning explains why.
- `registerRule()` no-op returns warning string `"openai-codex adapter does not support rules"`.
- `uninstall()` leaves no katana traces.
