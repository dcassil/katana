# Claude Code Adapter: Uninstall

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Implement `uninstall()` for the Claude Code adapter so an `install()` can be reversed. Removes the plugin scaffold, the katana entry from `.mcp.json`, and the marker block from `CLAUDE.md` while preserving any user content.

## Deliverables
- `uninstall()` method in `src/platform/claude-code/index.ts`.
- CLI wiring: `katana uninstall <platform>` in `src/cli/install.ts` (shared module).
- `tests/platform/claude-code/uninstall.test.ts`

## Architecture & Contracts
Steps:
1. Delete `<workspaceRoot>/.claude/plugins/katana/` recursively.
2. If `.claude/plugins/` is empty, remove it; if `.claude/` is otherwise empty, remove it.
3. Parse `<workspaceRoot>/.mcp.json`; delete `mcpServers.katana`; if `mcpServers` becomes empty, delete the key; if root object becomes empty, delete the file.
4. In `<workspaceRoot>/CLAUDE.md`, remove content from `<!-- katana:begin -->` through `<!-- katana:end -->` inclusive plus a single trailing newline; if file becomes empty (or whitespace-only), delete it.
5. Return `InstallReport` whose `files[]` lists each removed/edited path with `action: "updated"` (for edits) or `"created"` repurposed as `"removed"` — extend `WrittenFile.action` enum to `"created"|"updated"|"skipped"|"removed"` (update port and downstream tests in this task).

## Do / Don't
- DO preserve user content outside katana markers byte-for-byte.
- DO be idempotent (uninstalling twice is a no-op).
- DON'T error if a file is already absent — report `skipped`.
- DON'T touch foreign `mcpServers` entries.

## Acceptance
- After install -> uninstall, the workspace is byte-identical to pre-install state for non-marker files; marker files retain their non-katana content.
- Foreign `mcpServers.foo` entry survives uninstall.
- Double-uninstall is a no-op.
