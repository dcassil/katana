# Cursor Adapter: Rules and MCP Config

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Implement `CursorAdapter` (basic — covers universal commands via rules + MCP config; no platform-specific niceties). On install, writes `.cursor/rules/katana.mdc` and merges `.cursor/mcp.json`.

## Deliverables
- `src/platform/cursor/index.ts`
- `src/platform/cursor/templates/katana.mdc.tmpl`
- `src/platform/cursor/templates/mcp.json.tmpl`
- `tests/platform/cursor/install.test.ts`

## Architecture & Contracts
Files written under `<workspaceRoot>`:
- `.cursor/rules/katana.mdc` — Cursor MDC rule scoped `alwaysApply: true`, body documents the four universal commands and instructs the agent to call the katana MCP tools.
- `.cursor/mcp.json` — same shape as Claude Code's `.mcp.json` (merge-safe).

`registerCommand(spec)` -> appends a bullet to the `katana.mdc` body inside marker block. `registerRule(spec)` with `scope: "directory"` -> writes `.cursor/rules/<id>.mdc` with `globs: spec.globs`. `generateAgentDoc()` -> Cursor has no global agent doc; this method writes/updates the marker block in `katana.mdc` instead and returns its `WrittenFile`.

`.cursor/mcp.json` template:
```json
{
  "mcpServers": {
    "katana": { "command": "{{mcpCommand}}", "args": {{mcpArgs}}, "transport": "stdio" }
  }
}
```

Idempotency rules identical to Claude Code adapter (hash + skip).

## Do / Don't
- DO place the katana rule at `alwaysApply: true` so universal commands are always discoverable.
- DO merge-preserve other `mcpServers` entries.
- DON'T attempt to register Cursor "agent commands" (out of scope for MVP).
- DON'T write to host-level Cursor config (`~/.cursor/...`) — workspace-only.

## Acceptance
- Fresh repo install produces both files; golden-fixture match.
- Re-install: skipped.
- Existing `.cursor/mcp.json` with foreign server preserved.
- `InstallReport.commands` lists all four universal commands; `mcpRegistered: true`.
