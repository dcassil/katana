# Platform Adapter Smoke Test (Integration)

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
End-to-end integration test that exercises all three adapters against fresh temp directories using the CLI and validates the resulting workspace shape. Satisfies the initiative exit criterion: "At least one non-Claude adapter installed and functional for a smoke-test workflow."

## Deliverables
- `tests/integration/platform-smoke.test.ts`
- `tests/integration/fixtures/expected-claude-code/` (golden tree)
- `tests/integration/fixtures/expected-cursor/`
- `tests/integration/fixtures/expected-openai-codex/`

## Architecture & Contracts
For each platform:
1. Create a tmp dir; `runInstall(["install", platform, "--workspace", tmp, "--mcp-command", "npx", "--mcp-args", "-y,katana-mcp"])`.
2. Walk the resulting tree, normalize timestamps, compare to golden fixture.
3. Run install a second time; assert all `WrittenFile.action === "skipped"`.
4. Mutate one written file; run install again; assert it is repaired only with `--force`, otherwise reported as drift via `InstallReport.warnings`.
5. Run `uninstall()` directly via the adapter; assert tree returns to pre-install state (except agent-doc files like `AGENTS.md` that retain text outside markers).

Cross-adapter assertions:
- All three reports list the four universal command ids exactly.
- `mcpRegistered` true for `claude-code` and `cursor`, false for `openai-codex`.

## Do / Don't
- DO use `vitest`'s `tmpdir` helper; never write to the repo working tree.
- DO snapshot file contents, not just paths.
- DON'T spawn a real Claude Code or Cursor process — file shape is the contract.
- DON'T rely on network.

## Acceptance
- All three adapter cases pass.
- Idempotency assertion passes for each.
- Drift detection assertion passes for each.
- Uninstall cleanup passes for each.
