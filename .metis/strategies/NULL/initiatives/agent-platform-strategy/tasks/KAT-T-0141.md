# PlatformAdapter Port Interface and Types

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Define the `PlatformAdapter` interface and all supporting types in `src/platform/port.ts`. This is the contract every concrete adapter (Claude Code, Cursor, OpenAI/Codex) implements. No platform code in this task — types and JSDoc only.

## Deliverables
- `src/platform/port.ts` exporting: `PlatformAdapter` interface, `InstallOptions`, `InstallReport`, `CommandSpec`, `RuleSpec`, `AgentDocSpec`, `PlatformId`, `WrittenFile`.
- `tests/platform/port.types.test.ts` with `expectType<>`-style compile-only assertions to lock the shape.

## Architecture & Contracts
```ts
export type PlatformId = "claude-code" | "cursor" | "openai-codex";

export interface InstallOptions {
  workspaceRoot: string;          // absolute path; host repo root
  katanaRoot: string;             // absolute path; usually <workspaceRoot>/.katana
  mcpCommand: string;             // e.g. "npx katana-mcp"
  mcpArgs?: string[];
  dryRun?: boolean;               // when true, return report without writing
  force?: boolean;                // overwrite existing files
}

export interface WrittenFile { path: string; action: "created" | "updated" | "skipped"; bytes: number; }

export interface InstallReport {
  platform: PlatformId;
  files: WrittenFile[];
  mcpRegistered: boolean;
  commands: string[];             // command ids exposed
  warnings: string[];
}

export interface CommandSpec {
  id: string;                     // e.g. "katana-decompose"
  description: string;
  argsSchema?: object;            // JSON Schema (draft-7); optional
  handlerHint: { mcpTool: string; }; // 1:1 mapping to MCP tool name
}

export interface RuleSpec {
  id: string;                     // e.g. "katana-core"
  scope: "workspace" | "directory";
  body: string;                   // markdown
  globs?: string[];               // for directory-scoped rules
}

export interface AgentDocSpec {
  filename: string;               // e.g. "CLAUDE.md", "AGENTS.md"
  block: string;                  // markdown content to inject between katana markers
  markerStart: string;            // "<!-- katana:begin -->"
  markerEnd: string;              // "<!-- katana:end -->"
}

export interface PlatformAdapter {
  readonly id: PlatformId;
  install(opts: InstallOptions): Promise<InstallReport>;
  uninstall(opts: Pick<InstallOptions, "workspaceRoot" | "katanaRoot">): Promise<InstallReport>;
  registerCommand(spec: CommandSpec, opts: InstallOptions): Promise<WrittenFile[]>;
  registerRule(spec: RuleSpec, opts: InstallOptions): Promise<WrittenFile[]>;
  generateAgentDoc(spec: AgentDocSpec, opts: InstallOptions): Promise<WrittenFile>;
}
```
All filesystem writes MUST be idempotent (re-running `install` is a no-op when files match). Marker-bracketed regions are the only mutation strategy for shared files (`CLAUDE.md`, `AGENTS.md`).

## Do / Don't
- DO export every type as a named export.
- DO keep this file dependency-free (no Node `fs`, no MCP SDK imports).
- DON'T put concrete logic here.
- DON'T leak platform-specific fields onto the shared interface.
