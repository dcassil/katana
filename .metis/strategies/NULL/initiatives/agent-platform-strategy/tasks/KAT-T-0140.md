# ADR: Universal vs Platform-Specific Command Split

## Parent Initiative
KAT-I-0006 — Agent Platform Strategy Pattern.

## Goal
Decide and record (as ADR `KAT-A-0002` under `docs/adr/`) which katana commands every PlatformAdapter MUST surface ("universal") and which are optional / per-platform niceties. Lock command names, argument shapes, and the rule that universal commands map 1:1 onto MCP tools so any platform can fall back to raw MCP invocation.

## Deliverables
- `docs/adr/KAT-A-0002-universal-vs-platform-commands.md` (status: Accepted, dated 2026-05-09).
- Updated initiative file referenced from the ADR's "Drives" section.

## Architecture & Contracts
Universal command set (MUST be exposed by every adapter, identical names and semantics):
- `decompose <parent-short-code>` — wraps MCP `decompose_document`.
- `work <task-short-code>` — selects a task and starts the work loop on the host platform.
- `board [--level <level>] [--phase <phase>]` — wraps MCP `list_documents` + presents kanban view.
- `validate <short-code>` — wraps MCP `validate_document`.

Platform-specific (optional): hooks (Claude Code only), inline gate UI (Cursor), repo-scoped agent presets (Codex). Adapters MAY expose more but MUST NOT rename or alter the universal four.

Naming rule: in Claude Code these surface as slash commands `/katana-decompose`, `/katana-work`, `/katana-board`, `/katana-validate`. In Cursor as rule-triggered actions of the same names. In Codex as `AGENTS.md` documented invocations against the MCP server directly.

## Do / Don't
- DO restrict the universal set to four commands. Resist scope creep.
- DO require each universal command to map 1:1 to an MCP tool listed in KAT-A-0001.
- DON'T add `init` to the universal set — install/init is the adapter's `install()` method, not a runtime command.
- DON'T let any adapter rename the universal commands.

- ADR explicitly enumerates the four universal commands and their MCP tool mapping.
- ADR cross-links KAT-I-0006 and KAT-A-0001.
