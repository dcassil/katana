## Goal

## Architecture & Contracts

### `src/storage/markdown/index.ts`

```ts
import * as fs from "node:fs";
import * as path from "node:path";
import * as crypto from "node:crypto";
import matter from "gray-matter";
import type { Document, Frontmatter } from "../../types/document.js";

/** Deterministic field order for serialized frontmatter. */
const FIELD_ORDER: (keyof Frontmatter)[] = [
  "id", "level", "title", "short_code", "subtype",
  "parent", "story_id", "scaffold_task",
  "pass", "model_tier",
  "created_at", "updated_at",
  "archived", "tags", "exit_criteria_met", "phase",
  "blocked_by", "strategy_id", "initiative_id",
];

export function computeHash(content: string): string {
  return crypto.createHash("sha256").update(content, "utf8").digest("hex");
}

export function readMarkdown(filepath: string): Document {
  const raw = fs.readFileSync(filepath, "utf8");
  const parsed = matter(raw);
  const fm = parsed.data as Frontmatter;
  return {
    frontmatter: fm,
    body: parsed.content.replace(/^\n/, ""),
    filepath,
    file_hash: computeHash(raw),
  };
}

/** Serialize frontmatter in canonical order; trim and ensure trailing newline. */
export function serialize(fm: Frontmatter, body: string): string {
  const ordered: Record<string, unknown> = {};
  for (const k of FIELD_ORDER) {
    if (fm[k] !== undefined) ordered[k] = fm[k];
  }
  // gray-matter.stringify emits valid YAML frontmatter.
  const out = matter.stringify(body.endsWith("\n") ? body : body + "\n", ordered, {
    // gray-matter passes engine options through; default js-yaml is fine.
  });
  return out;
}

export function writeMarkdown(filepath: string, fm: Frontmatter, body: string): { file_hash: string; bytesWritten: number } {
  fs.mkdirSync(path.dirname(filepath), { recursive: true });
  const text = serialize(fm, body);
  fs.writeFileSync(filepath, text, "utf8");
  return { file_hash: computeHash(text), bytesWritten: Buffer.byteLength(text, "utf8") };
}
```

### Tests must cover

- Round-trip: `writeMarkdown` then `readMarkdown` yields identical frontmatter and body.
- Order: written file's first 5 lines after `---` are `id:`, `level:`, `title:`, `short_code:`, `subtype:` (skipping unset).
- `computeHash` matches hash of bytes on disk.
- Body without trailing newline is normalized to have one.

## Do / Don't

- **Do** reuse `gray-matter`; do not write a YAML emitter from scratch.
- **Do** keep `serialize` pure (no IO).
- **Don't** sort tags or `blocked_by` arrays — preserve caller order.
- **Don't** mutate the input frontmatter object.

## Completion Notes (2026-05-18)

Code at `src/storage/markdown/index.ts` was already present. One bug fixed:

- **Round-trip ambiguity for trailing newline**: the test suite has two mutually-tight cases — a body with trailing `\n` must round-trip with it, and a body without one must round-trip without it. `matter.stringify` always emits the same byte sequence for both inputs (it normalizes to one trailing `\n`), so the reader could not distinguish.
- **Fix**: serialize now signals the original-trailing-newline bit via a blank line between frontmatter and body. When body ends in `\n`, emit `---\n<yaml>---\n\n<body>` (blank line present). Otherwise emit `---\n<yaml>---\n<body>\n` (no blank line, normalizing `\n` appended). The reader detects this by looking at whether `parsed.content` starts with `\n`: if yes, strip the leading `\n` (preserving trailing); if no, strip the trailing `\n`.
