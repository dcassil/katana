## Goal

Implement `edit_document` as a strict literal search/replace within a document's markdown body.

## Deliverables

- `src/mcp/tools/edit_document.ts`

## Architecture & Contracts

### `src/mcp/tools/edit_document.ts`

```ts
import type { StoragePort } from "../../storage/port.js";
import type { Document, ShortCode } from "../../types/document.js";
import { TOOL_SCHEMAS } from "./schemas.js";

interface Input { short_code: ShortCode; search: string; replace: string; replace_all?: boolean; }

export const editDocumentTool = {
  ...TOOL_SCHEMAS.edit_document,
  handler: async (input: Input, ctx: { storage: StoragePort }): Promise<Document> => {
    if (input.search.length === 0) throw new Error("search must be non-empty");
    return ctx.storage.edit(input.short_code, input.search, input.replace, input.replace_all ?? false);
  },
};
```

### Storage contract recap (from KAT-T-0084)

`edit(short_code, search, replace, replace_all)`:
- Reads document body.
- If `search` not found in body, throw `Error("search string not found")`.
- If `replace_all = false`, replace first occurrence; else all.
- Frontmatter is **not** touched.
- `updated_at` is bumped; `file_hash` recomputed; row + file written in one transaction.

## Do / Don't

- **Do** treat `search` as a literal string, not a regex.
- **Do** error (not silent no-op) when `search` is absent.
- **Don't** apply the edit to frontmatter — only the markdown body.
- **Don't** introduce a `regex` mode; deferred.

## Acceptance

- Unit test: edit hits, edit miss throws, replace_all replaces all occurrences.
